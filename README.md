# BUPT 校园导航系统

本项目是“数据结构算法与课程设计”作业的基础数据模块。通过 Python 开发的交互式工具，完成了北京邮电大学西土城校区关键建筑、出入口及路口的坐标采集与物理建模。

## 📍 坐标系与参数说明

项目实现了从图像像素空间到物理米制空间的映射。为了便于后续算法计算，物理坐标系已做对齐处理。

| 参数 | 设定值 | 说明 |
| :--- | :--- | :--- |
| **比例尺 (Scale)** | 1.0 (约 0.5m) | 1 单位像素对应的实际物理距离 |
| **参考原点** | (220, 2700) | 图像像素坐标，作为物理坐标系的 (0,0) 点 |
| **Y轴校准** | 向上为正 | 计算公式：$Y_{real} = (PY_{origin} - PY_{click}) \times SCALE$ |

> **注**：由于地图经过偏移处理，物理坐标主要用于相对位置计算与拓扑网络构建。

## 📊 数据说明 (`campus_nodes.csv`)

采集到的数据共包含 240+ 个有效节点，涵盖了校园核心区域。

### 字段定义
- `Node_ID`: 节点的唯一识别码。
- `Name`: 节点名称（如：教一楼、教一楼_东门）。
- `Type`: 节点分类。
    - `Building`: 建筑中心点（逻辑位置）。
    - `Door`: 建筑出入口（路径导航的起终点）。
    - `Intersection`: 路口/转折点（拓扑网络的核心连接点）。
- `X / Y`: 物理坐标（单位：米），已转换为笛卡尔坐标系。
- `PX / PY`: 原始图像像素坐标。
- `Parent`: 隶属关系（门所属的建筑名称）。

## 🛠️ 采集工具说明 (`map_tool.py`)

工具基于 `Matplotlib` 和 `Pandas` 开发，具备以下特性：
1. **交互式采集**：鼠标左键点击拾取，终端输入属性。
2. **实时撤销**：支持键盘 `Z` 键一键撤销（同步删除视觉标记与内存数据）。
3. **数据恢复**：启动时自动加载旧 CSV 数据并在地图上重绘，支持增量采集。
4. **防御性编程**：自动校准 PX/PY，确保手动微调 $X, Y$ 后地图显示同步。

## ⚠️ 采集局限性声明
在数据采集过程中，针对复杂环境进行了如下简化处理：
- **简化建模**：物美、眼镜店等小型商业设施简化为单一坐标点。
- **开放空间**：小松林、时光广场等无明确边界区域暂未采集。
- **家属区**：由于建筑命名复杂且道路细碎，暂未纳入导航网络。
- **道路细节**：绿化带、草坪小径等非干道暂未标注。
