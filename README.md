# 📍 BUPT 校园导航系统 (BUPT Campus Navigation System)

本项目是“数据结构算法与课程设计”作业。通过 Python 构建了一个完整的校园路网拓扑模型，实现了基于 Dijkstra 算法的最短路径规划，并支持多入口建筑的智能搜索与可视化交互。

---

## 🚀 核心功能

1.  **智能路径查询**：支持直接输入建筑名称（如“南门”、“校医院”）。对于拥有多个入口的建筑，系统会自动计算并选择距离起点最近的门口（Door）作为目的地。
2.  **最短路径算法**：底层采用**堆优化 Dijkstra 算法**。通过优先队列将算法效率提升至 $O(E \log V)$，确保在校园 240+ 节点量级下实现瞬时导航。
3.  **图结构管理**：
    - **遍历**：支持广度优先遍历 (BFS)，可用于分析校园建筑的连通性及周边设施推荐。
    - **动态修改**：支持在交互菜单中实时添加、删除节点或道路，并支持数据持久化保存。
4.  **可视化输出**：导航结果将实时高亮标注在校园底图上，支持保存为 **PNG 高清格式**，并提供详细的文字路径描述。

---

## 🛠️ 技术实现

### 1. 建模逻辑与数据结构
* **存储结构**：使用 `pandas` 处理 CSV 数据。在程序运行时，数据被转化为**邻接表 (Adjacency List)**，结构为 `{Node_ID: {Neighbor_ID: Distance}}`。
* **多入口逻辑**：利用 `Parent` 字段将“门 (Door)”与“建筑 (Building)”关联。导航时，算法会将目标建筑的所有门视为“目标集合”，实现“单源多汇”的最短路径寻优。

### 2. 坐标映射原理
项目实现了从图像像素空间 ($PX, PY$) 到物理米制空间 ($X, Y$) 的线性映射。
* **比例尺 (Scale)**: 1.0 (约等于实际 0.5 米)。
* **参考原点**: 以图像像素 (220, 2700) 为物理坐标系 $(0,0)$ 点。
* **计算公式**: $Y_{real} = (PY_{origin} - PY_{click}) \times SCALE$。

## 📂 项目文件组成

| 文件名 | 作用描述 |
| :--- | :--- |
| **map_system.py** | **核心逻辑**。包含 Dijkstra 算法实现、多入口包装函数及交互式菜单。 |
| **map_tool.py** | **数据采集工具**。用于在 `map.jpg` 上点击取点并记录物理坐标。 |
| **build_graph.py** | **可视化调试工具**。用于展示当前所有节点分布及路网连通状态。 |
| **campus_nodes.csv** | 节点数据库。包含 240+ 处建筑、校门、路口及内部入口的坐标数据。 |
| **campus_edges.csv** | 道路数据库。存储节点间的物理距离权重，构成图的边集。 |
| **map.jpg** | 校园底图。作为所有坐标定位与路径绘制的参考背景。 |

## 🖥️ 使用指南

### 1. 环境准备
确保已安装必要的 Python 库：
```bash
pip install pandas matplotlib pillow numpy

2. 启动系统
直接运行主程序进入交互菜单：

Bash
python map_system.py
3. 操作流程
路径查询：选择功能 1，输入起点和终点（如“南门”、“教一楼”）。

查看结果：系统会打印路径文字指引，并弹出带有红线标注的可视化地图。

管理地图：若有新修道路，可通过功能 3 输入节点 ID 进行维护。

保存退出：选择功能 4 退出，所有对路网的修改将同步写入 CSV 文件。

⚠️ 注意事项
误差说明：由于路网连线取直线距离且地图存在投影偏移，导航距离仅供参考，实际请以校园路标为准。

中文显示：代码依赖 SimHei 字体。若在非 Windows 环境下运行，请修改代码中的 plt.rcParams 配置。
